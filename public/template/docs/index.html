Getting started #back to top

This template uses Bootstrap 4 CSS framework, a powerful tool to help you building great websites in a short time.

Template's directory structure

  • HTML files - HTML files are located in the root folder. These are index.html, tables.html, etc.
  • /css - CSS files including Bootstrap CSS, CSS for the components and main template stylesheets
  • /data - Sample data for some plugins
  • /icons-reference - Custom icons list and sources
  • /fonts - Font Awesome icon font
  • /img - Images for demo purposes. Mostly replaced by placeholder images in the download archive.
  • /js - Javascript files
  • /scss - SCSS source files for easier customization

CSS #back to top

Switching colour variant

The theme stylesheet is css/style.default.premium.css. Colour variants are named accordingly, to use a different colour variant, just replace link to the style.default.premium.css with e.g. style.blue.premium.css in the <head> element in HTML files.

Custom CSS code

I suggest placing your changes and CSS declarations in custom.css file. This way, you will keep your code separated and you can update the original template stylesheet if an updated is released.

SCSS #back to top

What is SASS and SCSS

This template contains its SCSS source codes too. These can be very handy when you would want to create your own custom colour variant, create a new button style and similar situations.

If you are completely new to SASS, check out their official website and download a compiler for your OS. SASS uses a bit different syntax than CSS but you should not have a problem working with it if you understand CSS.

File structure

The easies way to change theme colours, fonts and any CSS rules in general is by editing the SCSS files.

In the SCSS directory, there is a main file style.default.premium.scss or its colour variants. In these files, we just define which of the predefined colours will be used and the rest is done by core-premium.scss file. It takes control of all the necessary module includes. The modules (charts, forms, etc..) are located in the modules subdirectory.

_core-premium.scss (core file handling all the includes)
style.default.premium.scss (main file) 
style.blue.premium.scss (main file) 
style.green.premium.scss (main file) 
style.pink.premium.scss (main file) 
style.red.premium.scss (main file) 
style.sea.premium.scss (main file) 
style.violet.premium.scss (main file) 
_theme.scss (main Bootstrap components based on settings in the variable file)

-- <modules> (includes all partials included in the main stylesheets - style.default.scss, etc.)

---- _general.scss - generic styling of the template -buttons, header, footer, sections..
---- _mixins-custom.scss - my custom mixins
---- _mixins.scss - Bootstrap 4 mixins (included from the mixins directory)
---- _variables-custom.scss - file based on the Bootstrap 4 variables file.
---- _components.scss - styles for the components pages
---- _forms-premium.scss - styling of the premium elements in forms - datepickers, sliders, etc.
---- _forms.scss - generic form styling
---- _charts-premium.scss - Styling for the premium charts
---- _charts.scss - generic charts styling
---- _login.scss - Login page
---- _spinkit.scss - SpinKit preloaders
---- _tables.scss - generic tables styling
---- _demo.scss - styles for the demo purposes only (style switcher)

---- <mixins> (original Bootstrap 4 mixins)
---- <spinners> SpinKit spinners
---- <utilities> (original Bootstrap 4 ulilities)

JavaScript #back to top

Main JavaScript file

Main template file is called js/front.js and takes care of the following tasks. Most of them can be reconfigured there too.

  • Affixing bottom copyright section
  • Dropdowns fading in
  • Sidebar toggling
  • Search overlay
  • Forms' validation
  • Closing cards

As this premium version of Bootstrap Dashboard uses many 3rd party JavaScript plugins, they get included only on the pages where they are needed. The names of these scripts usually correspond to the HTML file name. E.g. components-calendar.js is needed on components-calendar.html page.
